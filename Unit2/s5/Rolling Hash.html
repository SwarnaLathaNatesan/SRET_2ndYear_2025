What is a Rolling Hash?

A rolling hash allows you to compute a hash value for a sliding window over a string in constant time per window, after an initial linear time computation.

H = (s[0]·p^(m-1) + s[1]·p^(m-2) + ... + s[m-1]·p^0) % M

Where:

  • p = base (prime number, e.g., 31 or 53)
  • M = large prime modulus (e.g., 1e9+7)
  • s = input string
  • m = length of the substring

Why Use Rolling Hash?

Rolling hashes enable efficient substring search by allowing O(1) hash updates when sliding the window, compared to O(m) for recalculating from scratch.

Collision Handling

Since different strings might have the same hash value (collision), it's important to verify matches by direct comparison when a hash match is found.
