Python's Collision Resolution

index = (hash(value) + probe_number) % table_size

When a collision occurs (two values hash to the same index), Python doesn't just fail. It uses a method called Open Addressing with a probing sequence to find the next available slot.

Linear Probing: If the initial index is occupied, Python checks the next index, and the next, until it finds an empty slot.

Example: If both 3 and 55 hash to index 1, but index 1 is occupied by 3, then 55 will be placed in the next available slot (index 2).

Lookup: When searching for 55, Python will check index 1 first, find 3, then continue probing until it finds 55 at index 2.
